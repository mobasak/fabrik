## How to Use

### Reviewing Plans

1. Open each plan file
2. Read through all issues
3. Add `[skip]` after items you want to skip:
   ```
   - [ ] [skip] **42.** Some issue I don't want to fix...
   ```

### Executing Plans

Say one of:
- **"Execute all plans"** - Run all 6 plans in order
- **"Execute plan 1"** or **"Execute core plan"** - Run specific plan
- **"Execute plans 1-3"** - Run a range

### Safety Guarantees

For each fix I will:

1. **Read the current code** before making changes
2. **Preserve function signatures** - No breaking API changes
3. **Use env vars with defaults** - Current behavior preserved
4. **Run existing tests** after changes
5. **Stop if tests fail** - Report before continuing

### Rollback

If anything breaks:
```bash
git diff HEAD~1  # See what changed
git checkout HEAD~1 -- <file>  # Revert specific file
```
